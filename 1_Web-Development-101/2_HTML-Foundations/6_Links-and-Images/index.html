Opening Links in New Tab

TLDR; New Tab Security For links to be opened in a new tab use both `noopener` & `noreferrer`. Documentation suggest that

Link to Odin Project about page w/ the following rel attribute (used to describe relation between current page and linked doc) values:

  • noopener: "... attribute ensures that a link opened in a new tab or window cannot interact with or access the original page. Without it, the new page can use JavaScript to manipulate the original page, which poses a security risk."
  • noreferrer: "attribute provides both privacy and security. It prevents the new page from knowing where the user came from (hiding the referrer) and also includes the behavior of noopener, preventing the new page from accessing the original page."
The rel attribute value noreferrer includes the functionality of noopener so must be sufficient on its own. These two attributes are solely for security/privacy and are used for preventing phishing attacks. Odin About Page
